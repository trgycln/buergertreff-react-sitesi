# 🚀 EÜR OTOMATIK VERİ AKTARIMI - ÖZET

## ✅ TAMAMLANDI

Mevcut **accounting_transactions** verileriniz otomatik olarak **accounting_eur_records** tablosuna dönüştürülecek!

---

## 📋 KURULUM (2 Adım)

### **1. Supabase'de Tablo Oluştur**
- Dosya: [`update_eur_records_table.sql`](update_eur_records_table.sql)
- Supabase SQL Editor → Kopyala → Yapıştır → RUN

### **2. Uygulamayı Başlat**
- Pom başlatılırken otomatik migration yapılır
- Konsolda: `✅ EÜR kaydı başarılı: X kayıt migre edildi`

---

## 📊 OTOMATİK İŞLEMLER

Tüm **transactions** otomatik olarak:

### **1️⃣ Buchungsnummer Alır**
```
EÜR-2026-00001
EÜR-2026-00002
EÜR-2025-00001
...
```

### **2️⃣ Dört Sphäre'ye Atanır (Otomatik)**

Kategori adına göre otomatik eşleştirme:

| Kategori İçerse | Sphäre | Örnek |
|---|---|---|
| Zins, Bank, Kredit | **Vermögensv** | "Banka Faizi" |
| Kurs, Event, Aktivität, Seminar | **Zweckbetrieb** | "Seminar Geliri" |
| Verkauf, Reklam, Shop | **Wirtschaftlich** | "Kaffee Satışı" |
| Diğer (Aidat, Bağış, vs.) | **Ideeller** | "Üye Aidatı" |

### **3️⃣ Verileri Yapılandırır**

```
from accounting_transactions → to accounting_eur_records
├── date → entry_date
├── description → vorgang (İşlem Açıklaması)
├── receipt_no → belegnummer
├── amount → amount_gross
├── category.name → sub_category & sphaere (otomatik)
└── file_no → stored_location
```

### **4️⃣ Yıl Bazında Sıralı Numara Verir**

Her mali yıl için ayrı sayaç:
```
2025: EÜR-2025-00001 ... EÜR-2025-00250
2026: EÜR-2026-00001 ... EÜR-2026-00150
```
